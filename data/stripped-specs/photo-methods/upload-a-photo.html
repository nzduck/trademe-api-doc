Upload a photo

Adds the photo to the authenticated user's list of photos. These photos can be used when selling.

Our photo retention policy is as follows:

  • Photos that were uploaded less than 10 days ago are not eligible for deletion.
  • Photos that are used in an active listing are not eligible for deletion.
  • Photos that were used in an auction that closed less than 45 days ago are not eligible for deletion.
All other photos may be deleted at our discretion.

It is recommended that a new photo be uploaded whenever you create a new listing. If you are relisting then uploading a new photo is not required, since you cannot relist a listing that closed more than 45 days ago.

URL: https://api.trademe.co.nz/v1/photos/Add.{file_format}
HTTP Method: POST
Requires Authentication? Yes
Permission Required: MyTradeMeWrite: Maintain your watchlist, answer questions and post comments.
Supported Formats: XML, JSON
Rate Limited? Yes

URL parameters

file_format Enumeration (required) The format of the response.

POST Data

<PhotoUploadRequest>

PhotoData String (required)

The actual image data which is base64 encoded.
FileName String (required)

A textual label for the image, which is intended to make it easier for the user to find the photo when creating a listing. If the file name looks like a path (with forward slash or backslash characters) all characters up to the last slash will be removed. The following characters are not allowed: " < > |
FileType String (required)

The file type of the image ("BMP", "GIF", "JPG", "JPEG" or "PNG"). Case is ignored.

Returns

<PhotoResponse>

The ID of the added photo.

Status Enumeration

The status of the photo operation.
PhotoId Integer

The ID of the photo.
PhotoUrls <PhotoUrl> or null

When uploading a new photo, contains a collection of URLs for this photo, one for each size.
